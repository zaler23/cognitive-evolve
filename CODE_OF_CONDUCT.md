# Code of Conduct

## Our pledge

We pledge to make participation in CognitiveEvolve welcoming, respectful, and harassment-free for everyone, regardless of background, identity, experience level, or viewpoint.

## Expected behavior

- Be respectful and constructive.
- Assume good intent, but accept correction when impact differs from intent.
- Keep criticism focused on ideas, code, evidence, and tradeoffs.
- Prefer specific, actionable feedback over dismissive comments.
- Respect privacy and do not expose secrets, credentials, private logs, or personal information.

## Unacceptable behavior

- Harassment, intimidation, threats, or sustained disruption.
- Personal attacks or discriminatory language.
- Publishing private information without permission.
- Sharing exploit details or security-sensitive information outside the process in `SECURITY.md`.
- Repeatedly ignoring maintainer boundaries or project governance.

## Enforcement

Maintainers may remove comments, close issues or pull requests, request behavior changes, or restrict participation when needed to protect the project community.

## Reporting

For conduct issues, contact the maintainers through a private maintainer channel when available. If only public GitHub issues are available, open a minimal issue requesting maintainer contact and do not include private personal information.

For security issues, follow `SECURITY.md` instead of this file.

## Scope

This code of conduct applies to project spaces including GitHub issues, pull requests, discussions, documentation, and project-related communication channels.
