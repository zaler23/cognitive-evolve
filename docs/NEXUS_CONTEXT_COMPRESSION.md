# Nexus Context Compression and Prompt Budgeting

Nexus keeps full evolution state locally, but model calls receive a bounded prompt view.

## Problem fixed

The exploration/recovery runtime persisted complete `population.json`, `archives.json`, round journals, verification traces, and candidate history correctly. The bug was that the model adapter reused those same full structures as provider payloads. As populations and archives grew, calls to ranking, critique, diagnosis, mutation planning, offspring generation, and synthesis could include repeated full candidate genomes, raw artifacts, full tool output, full verification traces, and repeated archive copies.

That made a single provider request grow into hundreds of thousands or millions of characters. The runtime therefore consumed quota by context volume rather than by simple call count.

## Current design

Local persistence remains complete:

- `population.json`
- `archives.json`
- `checkpoint.json`
- `candidate-journal.jsonl`
- `rounds/round-*.json`

Provider payloads are now compact views generated by `cognitive_evolve_runtime.nexus.prompt_view`.

The prompt view keeps:

- active / elite / rare / high-priority candidate summaries;
- candidate ids, generation, parent ids, fate, concise claim, core mechanism, scores, niches;
- short failure lessons and inherited genes;
- tool / verification status summaries instead of raw output;
- artifact previews plus hashes instead of full artifacts;
- archive counts and selected exemplars instead of full archive JSON;
- recent 1-2 round summaries instead of full history;
- project snapshot summaries instead of whole repository state.

The prompt view omits or summarizes:

- full `artifact` payloads;
- full `tool_results`;
- full `verification_trace`;
- full `mutation_history`;
- repeated archive copies of candidates;
- long raw command output;
- repeated historical round JSON.

## Budgets

The primary Nexus provider-payload cap is:

```bash
COGEV_NEXUS_PROMPT_MAX_CHARS=120000
```

Long-context Nexus requests may use:

```bash
COGEV_NEXUS_LONG_CONTEXT_MAX_CHARS=256000
```

If those are unset, the existing transport cap is honored:

```bash
COGEV_LLM_MAX_PROMPT_CHARS=120000
```

`cognitive_evolve_runtime.llm.transport` now also applies a final transport-level prompt bound before calling a provider. That layer is a safety net; Nexus should normally send already-compressed prompt views that fit the budget.

## Runtime observability

`NexusRuntime` records prompt-view metadata in `run-result.json` under:

```json
{
  "evolution": {
    "prompt_view_metadata": {
      "last": {
        "raw_payload_chars": 1840000,
        "compressed_payload_chars": 52000,
        "sent_payload_chars": 52000,
        "max_prompt_chars": 120000
      },
      "history": []
    }
  }
}
```

This lets operators distinguish between local state size and actual provider payload size.

## Important invariant

Exploration width is not reduced by prompt compression. Nexus may keep a large candidate population and rich archives locally. Compression only changes what is sent to the model on each call.
